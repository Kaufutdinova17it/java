##ЗАДАНИЕ 1: Дробь с кэшированием вычислений

**Решение**

- Реализован класс для работы с математическими дробями, включающий механизм кэширования десятичного значения.
- При первом вызове метода получения десятичного значения происходит вычисление, результат сохраняется в кэш. 
- При последующих вызовах возвращается закэшированное значение.
- Архитектурные решения
- Интерфейс FractionInterface определяет контракт для работы с дробями
- Класс Fraction реализует логику кэширования и нормализации дробей
- Принцип инкапсуляции: внутреннее состояние дроби защищено
- Автоматический сброс кэша при изменении числителя или знаменателя


##ЗАДАНИЕ 2: Паттерн Proxy для подсчета действий

**Решение**


- Реализованные паттерны
- Proxy (Заместитель): CountingCatProxy контролирует доступ к реальному объекту
- Interface Segregation: Чистый интерфейс Meowable с единственным методом
- Dependency Injection: Прокси получает реальный объект через конструктор
- Компоненты системы
- Интерфейс Meowable: определяет возможность мяукать
- Класс Cat: конкретная реализация мяукающего существа
- Класс CountingCatProxy: прокси для подсчета вызовов
- Утилитный класс MeowingUtils: работа со списком мяукающих объектов

##ЗАДАНИЕ 6: Обработка структур данных Queue

**Решение**
  
- Демонстрация различий между структурой данных "очередь" (FIFO - First In, First Out) и "стек" (LIFO - Last In, First Out).
- Алгоритм обратного вывода
- Проверка очереди на наличие элементов
- Последовательное извлечение элементов из очереди и помещение в стек
- Последовательное извлечение элементов из стека (обратный порядок)
- Вывод результатов в консоль
- Преимущества подхода
- Временная сложность: O(n) для n элементов
- Пространственная сложность: O(n) для временного хранения
- Сохранение исходной очереди: операция не разрушает исходные данные
- Универсальность: работает с любыми типами данных

##ЗАДАНИЕ 7: Применение Stream API

**Решение** 

- Подзадание 7.1: Обработка геометрических точек
- Цель: Преобразование набора точек в упорядоченную ломаную линию
- Операции Stream API:
- distinct() — удаление дубликатов точек
- sorted() — сортировка точек по координате X
- map() — преобразование отрицательных Y в положительные
- collect() — сбор результатов в список
- Бизнес-логика:
- Устранение избыточных точек
- Упорядочивание для визуализации
- Нормализация координат

Подзадание 7.2: Группировка данных из файла

**Решение**

- Процесс обработки:
- Чтение файла построчно
- Разделение строк по формату "имя:номер"
- Нормализация имен (первая буква заглавная, остальные строчные)
- Фильтрация некорректных записей
- Группировка по номерам с формированием списков имен
