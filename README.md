**Задача 1: Обобщенная коробка (Box<T>)**
--Решение
-Хранение: В классе объявлено приватное поле item типа T для хранения объекта.
-Помещение (put):

-Проверяется, не равен ли item null.

-Если коробка уже содержит объект, выбрасывается исключение.

-В противном случае объект сохраняется в поле item.

-Извлечение (get):

-Сохраняется значение item во временную переменную.

-Поле item устанавливается в null.

-Возвращается временная переменная.

-Проверка пустоты (isEmpty): Возвращает true, если item == null.



Задача 2: Хранилище без null (Storage<T>)
Цель: Создать неизменяемый обобщенный контейнер, возвращающий заданное значение по умолчанию при отсутствии основного.

Условие:
Создать класс Storage<T>, который:

Принимает основной объект и альтернативное значение при создании.

Является неизменяемым (все поля final).

Метод get() возвращает основной объект, если он не null, иначе — альтернативу.

Принцип работы:

Инициализация: В конструкторе устанавливаются два поля: item (основной объект) и alternative (значение по умолчанию).

Неизменяемость: Поля объявлены как final, что гарантирует их инициализацию только в конструкторе.

Получение значения (get):

Проверяется условие item != null.

Если истина — возвращается item.

Если ложь — возвращается alternative.

Типобезопасность: Все операции выполняются с типом T, указанным при создании экземпляра.

Реализация и демонстрация:
Выполнено четыре сценария:

Числа с null: Storage<Integer>(null, 0) → возвращает 0.

Числа со значением: Storage<Integer>(ввод_пользователя, -1) → возвращает введенное число.

Строки с null: Storage<String>(null, "default") → возвращает "default".

Строки со значением: Storage<String>(ввод_пользователя, "hello world...") → возвращает введенную строку.

Задача 3: Обобщенные методы (класс GenericUtils)
3.1. Метод map (преобразование)
Условие: Реализовать метод, применяющий заданную функцию к каждому элементу списка и возвращающий новый список результатов.

Принцип работы:

Параметризация: Метод объявлен как <T, R> List<R> map(...), где:

T — тип элементов входного списка

R — тип элементов выходного списка

Алгоритм:

Создается новый пустой список ArrayList<R> для результатов.

Для каждого элемента item в исходном списке:

Вызывается function.apply(item), получая результат типа R.

Результат добавляется в новый список.

Возвращается заполненный список результатов.

Примеры использования:

Список строк → список их длин (функция: String::length).

Список целых чисел → список их абсолютных значений (функция: Math::abs).

Список массивов чисел → список максимальных элементов каждого массива.

3.2. Метод filter (фильтрация)
Условие: Реализовать метод, возвращающий новый список, содержащий только элементы, удовлетворяющие заданному предикату.

Принцип работы:

Параметризация: Метод объявлен как <T> List<T> filter(...) с одним type parameter.

Алгоритм:

Создается новый пустой список ArrayList<T>.

Для каждого элемента item в исходном списке:

Вызывается predicate.test(item).

Если возвращается true, элемент добавляется в новый список.

Возвращается отфильтрованный список.

Примеры использования:

Отфильтровать строки длиной менее 3 символов.

Отфильтровать положительные числа из списка целых.

Отфильтровать массивы целых чисел, не содержащие положительных элементов.

3.3. Метод reduce (свёртка)
Условие: Реализовать метод, последовательно применяющий бинарный оператор к элементам списка для получения единственного результата. Гарантировать корректную работу с пустым списком.

Принцип работы:

Параметризация: Метод объявлен как <T> T reduce(...).

Алгоритм:

Если список null или пуст, сразу возвращается значение identity.

Инициализируется переменная result значением identity.

Для каждого элемента item в списке:

Вызывается operator.apply(result, item).

Результат становится новым значением result.

Возвращается итоговое значение result.

Гарантии: При пустом списке возвращается identity, что исключает NullPointerException.

Примеры использования:

Конкатенация списка строк в одну (оператор: (s1, s2) -> s1 + s2, identity = "").

Суммирование списка целых чисел (оператор: (a, b) -> a + b, identity = 0).

Подсчёт общего количества элементов во вложенных списках целых чисел.

3.4. Метод collect (коллекционирование)
Условие: Реализовать метод, который накапливает элементы исходного списка в произвольную результирующую коллекцию, используя предоставленные поставщик (Supplier) и аккумулятор (BiConsumer).

Принцип работы:

Параметризация: Метод объявлен как <T, R> R collect(...), где:

T — тип элементов входного списка

R — тип результирующей коллекции

Алгоритм:

Вызывается supplier.get() для создания пустой коллекции типа R.

Для каждого элемента item в списке:

Вызывается accumulator.accept(collection, item) для добавления элемента в коллекцию.

Возвращается заполненная коллекция.

Гибкость: Можно создавать любые структуры данных (List, Set, Map и т.д.).
